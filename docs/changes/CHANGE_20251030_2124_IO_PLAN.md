
### Design Pattern Extracted:
1. **contracts.py** - Pure interfaces, no implementation

3. **base.py** - Abstract classes + concrete utilities + registry
4. **errors.py** - Domain-specific exceptions
5. **__init__.py** - Clean public API

**Key Insight:** base.py contains BOTH abstract classes AND utilities (CodecRegistry, adapters)

## ?? Phase 3: Modular Paging System

### Current Implementation Issues:
- ? Hardcoded byte/line paging logic in PagedFileSource
- ? Not configurable or replaceable
- ? No strategy pattern

### Proposed Modular Paging (Strategy Pattern):

```python
# file/contracts.py
class IPagingStrategy(Protocol):
    """Strategy for how to page through data."""
    def read_page(self, source, page: int, page_size: int) -> Any: ...
    def iter_pages(self, source, page_size: int) -> Iterator: ...

# file/base.py
class APagingStrategy(ABC):
    """Base paging strategy."""
    @abstractmethod
    def read_page(self, source, page: int, page_size: int): ...

# file/paging/
+-- byte_paging.py      # BytePagingStrategy
+-- line_paging.py      # LinePagingStrategy
+-- record_paging.py    # RecordPagingStrategy (for CSV, JSONL)
+-- (future) smart_paging.py  # SmartPagingStrategy (adaptive)
+-- registry.py         # PagingStrategyRegistry

# file/paged_source.py
class PagedFileSource:
    def __init__(self, path, mode='rb', paging_strategy=None):
        self._strategy = paging_strategy or auto_detect_strategy(mode)
        # Strategy is PLUGGABLE and REPLACEABLE!
```

**Benefits:**
- ? Extensible: Add new strategies without touching PagedFileSource
- ? Configurable: Users can provide custom strategies
- ? Testable: Mock strategies for testing
- ? Performance: Optimize strategies independently

## ?? Phase 4: Extensible Archive System

### Current Implementation Issues:
- ? Hardcoded format logic in Archive class
- ? Cannot add 7z, RAR without modifying Archive
- ? No registry pattern like codec

### Proposed Registry-Based Archive (Like Codec!):

```python
# archive/contracts.py
class IArchiveFormat(Protocol):
    """Archive format handler interface."""
    format_id: str
    file_extensions: List[str]
    
    def create(self, files, output, **opts): ...
    def extract(self, archive, output_dir, **opts): ...
    def list_contents(self, archive): ...

# archive/formats/registry.py
class ArchiveRegistry:
    """Registry for archive formats (LIKE CodecRegistry!)."""
    def register(self, format_class): ...
    def get_by_extension(self, path): ...
    def get_by_id(self, format_id): ...

# archive/formats/zip.py
@register_archive_format
class ZipArchiver:
    format_id = "zip"
    file_extensions = [".zip", ".jar", ".war"]
    
    def create(self, files, output, **opts): ...
    def extract(self, archive, output_dir, **opts): ...

# archive/formats/tar.py
@register_archive_format
class TarArchiver:
    format_id = "tar"
    file_extensions = [".tar", ".tar.gz", ".tar.bz2", ".tar.xz"]
    ...

# archive/archive.py
class Archive:
    """Facade over archive registry (LIKE get_codec!)."""
    
    @staticmethod
    def from_file(path):
        """Auto-detect format from extension."""
        archiver = get_archiver_for_file(path)
        return archiver
```

**Benefits:**
- ? Extensible: Add new formats via registry
- ? Auto-detection: Like codec system
- ? No hardcoding: Format logic separated
- ? Consistent API: Same pattern as codec

## ?? Design Patterns Applied

### 1. **Strategy Pattern** (Paging)
- Pluggable paging strategies
- Can swap byte/line/record paging at runtime
- Future: SmartPagingStrategy (auto-select best)

### 2. **Registry Pattern** (Archive & Compression)
- Like CodecRegistry
- Auto-detection from file extension
- Decorator-based registration (@register_archive_format)

### 3. **Facade Pattern** (Archive, Compression)
- Simple API hiding registry complexity
- Archive.from_file() auto-detects format
- Compression.compress() auto-selects algorithm

### 4. **Factory Pattern** (Everywhere)
- from_file() static methods create appropriate instances
- Factories use registries for lookup
