## 🔧 Test Report Generation

### Automatic Generation

The main test runner (`tests/runner.py`) automatically generates TEST_* reports for:

**When:**
- Before version releases
- At project milestones
- After major refactoring
- Weekly full suite runs on CI
- Manual trigger with `--generate-report` flag

**Format:** `TEST_YYYYMMDD_HHMM_DESCRIPTION.md`

**Content:**
1. Test plan (scope, objectives, environment)
2. Results summary (pass/fail, coverage, performance)
3. Detailed test output (all layers, all tests)
4. Coverage analysis and trends
5. Issues found (if any)
6. Quality validation metrics

### Manual Creation

For special test campaigns:

```bash
# Run tests and manually create report
python tests/runner.py > test_output.txt
# Then create TEST_* report using the project test report template in this folder.
```

Use `TEMPLATE.md` in this folder for consistent formatting.

## 📖 Report Contents

Each TEST_* report includes:

**1. Test Plan:**
- What was tested
- Test objectives
- Test environment
- Test categories included

**2. Results Summary:**
- Overall pass/fail status
- Test execution summary table
- Coverage analysis
- Performance metrics

**3. Detailed Output:**
- Full test execution log
- Layer-by-layer results
- Module-by-module results
- Individual test results

**4. Quality Validation:**
- Security tests status
- Performance tests status
- Coverage trends
- Issues found

**5. Historical Context:**
- Comparison with previous runs
- Coverage evolution
- Trend analysis
