## 🎯 Planning Workflow

**1. Create Plan:**
- Read [REF_PLAN.md](../../REF_PLAN.md)
- Determine plan type (DEV, TEST, DOCS, PROJECT, MIXED)
- Use naming convention: `PLAN_YYYYMMDD_HHMM_DESCRIPTION.md`
- Follow template for plan type
- Add entry to [logs/SUMMARY_PLAN.md](../SUMMARY_PLAN.md)

**2. Execute Plan:**
- Update status: 🔵 Planned → 🟢 In Progress
- Check off tasks as completed
- Document decisions and changes
- Update logs/SUMMARY_PLAN.md with progress

**3. Complete Plan:**
- Update status: 🟢 In Progress → ✅ Complete
- Fill out completion report in plan
- Create CHANGE_*.md or PROJECT_*.md document
- Update logs/SUMMARY_PLAN.md with outcomes
- Link deliverables

**4. Archive (Optional):**
- For very old completed plans
- Move to `plans/_archive/` directory
- Update this index

## 📚 Planning Best Practices

**From REF_PLAN.md:**

1. **Think Before You Write**
   - Good planning saves 10x time in execution

2. **Be Specific**
   - Vague plans → vague results
   - Define measurable success criteria

3. **Follow eXonware 5 Priorities**
   - Security → Usability → Maintainability → Performance → Extensibility

4. **Use Checklists**
   - Break work into verifiable tasks
   - Track progress visibly

5. **Link to Guides**
   - DEV plans → GUIDE_DEV.md
   - TEST plans → GUIDE_TEST.md
   - DOCS plans → GUIDE_DOCS.md
   - PROJECT plans → GUIDE_PROJECT.md

6. **Update Progress**
   - Keep status current
   - Document blockers
   - Adjust as needed

7. **Document Outcomes**
   - Fill completion reports
   - Capture lessons learned
   - Link to deliverables
